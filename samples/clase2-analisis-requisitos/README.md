# Clase 2: Análisis de Requisitos - Funcionales vs No Funcionales

## 📋 Descripción General

Esta clase está diseñada para desarrolladores que necesitan dominar el análisis de requisitos en proyectos de software empresarial. A través de ejercicios prácticos con contexto FinTech, aprenderás a distinguir, documentar y gestionar requisitos funcionales y no funcionales.

**Contexto:** Sistema de procesamiento de pagos digitales  
**Enfoque:** Metodología práctica con ejemplos reales  
**Duración:** 3-4 horas (según nivel)

---

## 🎯 Objetivos de Aprendizaje

### Al completar esta clase podrás:
- ✅ Distinguir entre requisitos funcionales y no funcionales
- ✅ Convertir descripciones informales en casos de uso estructurados
- ✅ Crear user stories siguiendo metodología ágil
- ✅ Especificar requisitos no funcionales medibles
- ✅ Establecer matrices de trazabilidad y dependencias
- ✅ Aplicar técnicas de elicitación de requisitos

## 🎨 Estructura de Ejercicios

### Ejercicio 1: Identificación de Tipos de Requisitos
**Objetivo:** Distinguir entre requisitos funcionales y no funcionales

| Simple | Avanzado |
|--------|----------|
| ✨ 15 requisitos básicos para clasificar | 🎯 30 requisitos complejos de múltiples stakeholders |
| ✨ Preguntas guía paso a paso | 🎯 Análisis de conflictos y dependencias |
| ✨ Ejemplos cotidianos | 🎯 Contexto financiero y regulatorio |

### Ejercicio 2: Casos de Uso y User Stories
**Objetivo:** Convertir necesidades en especificaciones técnicas

| Simple | Avanzado |
|--------|----------|
| ✨ Identificar usuarios y acciones básicas | 🎯 Cases de uso formales con flujos de excepción |
| ✨ Plantillas pre-completadas | 🎯 User stories con criterios de aceptación BDD |
| ✨ Ejemplos de un solo usuario | 🎯 Múltiples actores y integraciones complejas |

### Ejercicio 3: Especificaciones No Funcionales
**Objetivo:** Definir requisitos de calidad medibles

| Simple | Avanzado |
|--------|----------|
| ✨ 4 categorías básicas (velocidad, seguridad, etc.) | 🎯 17 NFRs con métricas específicas |
| ✨ Conceptos explicados con analogías | 🎯 Estándares industriales (PCI DSS, GDPR) |
| ✨ Ejemplos prácticos simples | 🎯 Checklist de validación profesional |

### Ejercicio 4: Trazabilidad y Dependencias
**Objetivo:** Organizar y conectar todos los requisitos

| Simple | Avanzado |
|--------|----------|
| ✨ Tabla básica de conexiones | 🎯 Matriz de trazabilidad completa |
| ✨ Dependencias lineales simples | 🎯 Análisis de impacto y gestión de cambios |
| ✨ Organización en Excel | 🎯 Herramientas profesionales (Jira, Confluence) |

## 📈 Criterios de Evaluación

### Nivel Simple - Has dominado cuando:
- [ ] Puedes clasificar requisitos sin ayuda
- [ ] Entiendes la diferencia entre QUÉ y CÓMO
- [ ] Puedes escribir casos de uso básicos
- [ ] Organizas requisitos en tablas simples

### Nivel Avanzado - Has dominado cuando:
- [ ] Manejas stakeholders múltiples
- [ ] Defines NFRs con métricas específicas
- [ ] Creas matrices de trazabilidad completas
- [ ] Analizas dependencias e impactos
